# Help = Markdown manpages under `docs/man/`, echoed verbatim — the reference model for the sibling plugins

The sibling plugins' `help.py` shows only a skill's intro paragraph because there is no per-skill source with flags — a missing-source bug, not a formatting one. The fix chosen here, intended as the reference model to retrofit `kntnt-code-skills` and `kntnt-text-skills`: the single source of truth is `docs/man/<skill>.md`, a full manpage in **Markdown** (`NAME`, `SYNOPSIS`, `DESCRIPTION`, `OPTIONS`, `EXAMPLES`, `FILES`) — Markdown because the README links these pages and GitHub renders them; `SYNOPSIS` fenced and `OPTIONS` as a table so the monospace-sensitive parts survive.

**The tool echoes, it does not render**: Claude Code renders GitHub-flavoured Markdown in the terminal, so `help.py` needs no wrapping or alignment — no arg → overview (`plugin.json` blurb + each manpage's `NAME` line); `<skill>` → echo the manpage verbatim; unknown → an error line. Two entry points reach one source: `/kntnt-wp-skills:help [skill]` and a per-skill help-gate (each `SKILL.md`'s first step handles `help` / `--help` / `-h`). The README links to `docs/man/*` rather than restating usage. Because the script reads only `plugin.json` + `docs/man/*`, it drops into the sibling plugins unchanged.
